💡 El LED nervioso

Solución completa: reto base + ampliación con pulsador.

🧠 Idea de la solución

El reto base es la introducción canónica al for: parpadear N veces con un único bloque de código. La ampliación combina el for de hoy con el if de S02, aprovechando que el loop() se repite infinitamente para producir el "parpadeo lento por defecto".

🎯 La clave didáctica: el loop() ya hace de "bucle infinito" por sí mismo. No hace falta meter el parpadeo lento en otro bucle. Si el código del parpadeo lento está dentro del loop(), se ejecuta indefinidamente sin esfuerzo adicional. El if simplemente interrumpe ese parpadeo lento con 10 rápidos cuando se detecta el pulsador.

💻 Reto base — Código completo

void setup() { pinMode(13, OUTPUT); } void loop() { for (int i = 0; i < 5; i++) { digitalWrite(13, HIGH); delay(300); digitalWrite(13, LOW); delay(300); } delay(2000); // pausa larga entre tandas }

Cinco parpadeos a 300ms (600ms cada parpadeo completo = 3 segundos totales), luego pausa de 2 segundos, y el loop() vuelve a empezar.

💻 Ampliación — Código completo

void setup() { pinMode(13, OUTPUT); pinMode(7, INPUT); } void loop() { if (digitalRead(7) == HIGH) { // Modo nervioso: 10 parpadeos rápidos for (int i = 0; i < 10; i++) { digitalWrite(13, HIGH); delay(100); digitalWrite(13, LOW); delay(100); } } // Parpadeo lento por defecto (siempre se ejecuta una vez por vuelta del loop) digitalWrite(13, HIGH); delay(500); digitalWrite(13, LOW); delay(500); }

🔍 Cómo se lee el código

  • Cada vuelta del loop() comprueba si el pulsador está pulsado.
  • Si NO está pulsado (caso por defecto): el if es falso, se salta el for y va directamente al parpadeo lento. Después vuelve al inicio del loop(), vuelve a comprobar el pulsador, y vuelve a hacer otro parpadeo lento. Y así infinitamente.
  • Si SÍ está pulsado: entra en el for y hace 10 parpadeos rápidos seguidos (2 segundos en total). Cuando acaba, sale del if y ejecuta UN parpadeo lento. Luego vuelve al principio del loop(). Si el alumno ha soltado el botón, se mantiene en lentos.

⚠️ Errores típicos del alumnado

  • Olvidar el segundo delay() dentro del for: hacen digitalWrite(HIGH) + delay + digitalWrite(LOW) y se olvidan del último delay. El LED parece encendido permanentemente porque el siguiente HIGH ocurre inmediatamente.
  • Confundir i < 5 con i <= 5: con i <= 5 se repite 6 veces (i = 0, 1, 2, 3, 4, 5). Pueden confundirse de uno arriba o uno abajo.
  • Meter el parpadeo lento DENTRO del if: entonces el LED solo parpadea (rápido o lento) cuando se pulsa el botón. Sin pulsar, el LED estará apagado y el loop() no hace nada visible. Hay que insistir en que el parpadeo lento va FUERA del if.
  • Esperar que el cambio sea instantáneo: los alumnos pulsan el botón y se quejan de que "no responde rápido". Aquí toca explicar la limitación del delay() y aceptarla como aprendizaje real. Lo notarán especialmente con el parpadeo lento de 500 ms.

🎓 Variantes válidas que pueden traer los alumnos

Variante Validez Comentario
Otros valores de delay (250/250, 400/400 para el lento) ✅ Válido Mientras la diferencia entre lento y rápido sea clara.
for (int i = 1; i <= 10; i++) en vez de i = 0; i < 10 ✅ Válido Misma cantidad de iteraciones (10), aunque la convención habitual en C++ es empezar en 0.
Usar else con el parpadeo lento dentro ⚠️ Funciona pero discutir Funciona igual, pero rompe la elegancia de que el parpadeo lento es el "comportamiento por defecto". Mejor sin else.
Variable bool para guardar estado ❌ Innecesario Algunos avanzados intentarán esto. No hace falta: el loop() ya hace de "estado natural".

🎓 Estructuras usadas en esta solución

📌 Verificación pedagógica: esta solución usa solo estructuras vistas hasta esta sesión:
  • pinMode, digitalWrite, digitalRead, delay (S01)
  • if (S02)
  • for (concepto principal de S04)
No se han adelantado: while, millis(), funciones propias, arrays, ni operadores lógicos (AND/OR/NOT). El uso de bool tampoco aparece (aunque S03 lo introdujo) porque no se necesita.
