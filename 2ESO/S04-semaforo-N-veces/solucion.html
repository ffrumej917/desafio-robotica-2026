S04 — Semáforo de N ciclos

2º ESO · Tinkercad · Test de luces (FOR anidado) + semáforo de 5 ciclos

El reto de ampliación era…

Antes de los 5 ciclos del semáforo, hacer un test de luces: cada LED parpadea 3 veces para confirmar que está bien cableado. Después, ya sí, los 5 ciclos del semáforo normal.

🧠 Idea de la solución

🎯 La idea clave: el reto base usaba un bucle FOR (los 5 ciclos del semáforo). La ampliación añade tres bucles FOR más antes del semáforo, uno por cada LED del test. Total: 4 bucles FOR seguidos dentro de setup().

El test de luces es lineal: primero parpadea el rojo 3 veces, luego el ámbar 3 veces, luego el verde 3 veces. Después arranca el semáforo.

💻 Código completo

Arduino C++ 
const int rojo  = 11;
const int ambar = 12;
const int verde = 13;

void setup() {
    pinMode(rojo,  OUTPUT);
    pinMode(ambar, OUTPUT);
    pinMode(verde, OUTPUT);

    // === TEST DE LUCES: cada LED parpadea 3 veces ===
    for (int i = 0; i < 3; i++) {
        digitalWrite(rojo, HIGH);  delay(200);
        digitalWrite(rojo, LOW);   delay(200);
    }
    for (int i = 0; i < 3; i++) {
        digitalWrite(ambar, HIGH); delay(200);
        digitalWrite(ambar, LOW);  delay(200);
    }
    for (int i = 0; i < 3; i++) {
        digitalWrite(verde, HIGH); delay(200);
        digitalWrite(verde, LOW);  delay(200);
    }

    delay(500);  // pequeña pausa antes del semáforo

    // === SEMÁFORO: 5 ciclos completos ===
    for (int i = 0; i < 5; i++) {
        digitalWrite(rojo,  HIGH); delay(1000); digitalWrite(rojo,  LOW);
        digitalWrite(ambar, HIGH); delay(500);  digitalWrite(ambar, LOW);
        digitalWrite(verde, HIGH); delay(1000); digitalWrite(verde, LOW);
        digitalWrite(ambar, HIGH); delay(500);  digitalWrite(ambar, LOW);
    }
}

void loop() {
    // Vacío: todo el espectáculo ocurre en setup().
}

🔍 Cómo se lee el código

  • Cada uno de los tres primeros for hace que un LED parpadee 3 veces a 200 ms (200 encendido + 200 apagado).
  • Tras un delay(500) de descanso, viene el cuarto for con los 5 ciclos del semáforo (idéntico al reto base).
  • Al terminar todos los for, setup() acaba y el programa entra en loop(), que está vacío → los LEDs quedan apagados.

⚠️ Errores típicos del alumnado

  • Reutilizan la variable i sin redeclararla. En cada for hay que volver a poner int i, no basta con escribir for (i = 0; …).
  • Olvidan el delay entre HIGH y LOW al apagar el LED. Resultado: parece que el LED ni siquiera se enciende (porque se apaga inmediatamente).
  • Meten los 4 for dentro de loop() en lugar de setup(). Resultado: el test + semáforo se repite eternamente.
  • Cambian los pines y se les olvida actualizar pinMode(). Si conectan el rojo al pin 9 pero pinMode(11, OUTPUT), el LED rojo no se enciende.
📌 Verificación pedagógica: esta solución usa solo conceptos ya vistos hasta S04:
  • setup() / loop() (S01)
  • pinMode, digitalWrite, delay (S01)
  • const int y nombres simbólicos de pines (S01-S02)
  • for (int i = 0; i < N; i++) (concepto principal S04)
No se han adelantado funciones propias, arrays, condicionales if ni ningún otro concepto pendiente para sesiones siguientes.
