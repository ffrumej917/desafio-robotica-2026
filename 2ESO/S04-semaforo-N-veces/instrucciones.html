🚦 Semáforo de N ciclos

🧠
ENTENDER — Bucle FOR con número de repeticiones
⏱ 10 minutos
1

Hasta ahora todo el código iba dentro de void loop(), que se ejecuta una y otra vez sin parar. Eso es perfecto para un semáforo eterno, pero ¿y si queremos que el semáforo haga solo 5 ciclos y se apague? Necesitamos algo que tenga un final programado.

📌 Esa estructura es el bucle FOR: for (int i = 0; i < N; i++) { ... }. Le decimos cuántas veces N y el bloque de dentro se ejecuta exactamente esas veces.
2

El FOR tiene tres partes entre paréntesis separadas por punto y coma:

  • int i = 0 — inicialización: crea un contador i que empieza en 0.
  • i < N — condición: mientras se cumpla, el bucle sigue.
  • i++ — paso: al final de cada vuelta, suma 1 al contador.
💡 Lectura mental: «empieza con i=0, mientras i sea menor que N, ejecuta el bloque y suma 1 a i». Cuando i llega a N, el bucle termina y el programa sigue con lo de fuera.
3

Lo importante de hoy: pondremos el FOR dentro de setup(), no de loop(). Como setup() se ejecuta una sola vez al arrancar la placa, el semáforo hará sus 5 ciclos al encender Arduino y luego se quedará apagado. Si lo metieras en loop() haría 5 ciclos, breve pausa, otros 5 ciclos, breve pausa… eternamente.

✏️
DISEÑAR — Planifica el circuito y los tiempos
⏱ 10 minutos
4

El circuito tiene 3 LEDs en paralelo, uno por color, cada uno con su resistencia de 220 Ω en serie. Comparten todos el mismo GND.

🔴 LED rojo: ánodo → resistencia 220 Ω → pin 11 · cátodo → GND.
🟡 LED ámbar: ánodo → resistencia 220 Ω → pin 12 · cátodo → GND.
🟢 LED verde: ánodo → resistencia 220 Ω → pin 13 · cátodo → GND.
(En el ánodo va el patilla larga del LED; en el cátodo, la corta.)
⚠️ Sin las resistencias de 220 Ω, los LEDs se queman casi inmediatamente. Cada LED necesita la suya, no se puede compartir una.
5

Antes de tocar código, escribe en tu ficha la tabla de tiempos de un ciclo completo del semáforo:

OrdenLED encendidoDuración
1🔴 Rojo1 segundo
2🟡 Ámbar0,5 segundos
3🟢 Verde1 segundo
4🟡 Ámbar0,5 segundos
📐 Un ciclo dura 3 segundos. Si hacemos 5 ciclos: 5 × 3 = 15 segundos en total antes de que el semáforo se apague.
💻
PROGRAMAR — Monta y codifica en Tinkercad
⏱ 30 minutos
6

Abre tinkercad.com → Circuits → Crear nuevo circuito. Monta los 3 LEDs con sus resistencias según la descripción del paso 4.

7

Abre el editor de código (botón Código arriba a la derecha), cambia a Texto y escribe este programa:

const int rojo = 11;
const int ambar = 12;
const int verde = 13;
 
void setup() {
pinMode(rojo, OUTPUT);
pinMode(ambar, OUTPUT);
pinMode(verde, OUTPUT);
 
// 5 ciclos completos del semáforo
for (int i = 0; i < 5; i++) {
digitalWrite(rojo, HIGH); delay(1000); digitalWrite(rojo, LOW);
digitalWrite(ambar, HIGH); delay(500); digitalWrite(ambar, LOW);
digitalWrite(verde, HIGH); delay(1000); digitalWrite(verde, LOW);
digitalWrite(ambar, HIGH); delay(500); digitalWrite(ambar, LOW);
}
}
 
void loop() {
// Vacío: el semáforo solo se ejecuta una vez al arrancar.
}
💡 La variable i es interna del bucle: vale 0 en la primera vuelta, 1 en la segunda, 2 en la tercera… y el bucle termina cuando i intenta valer 5 (porque ya no se cumple i < 5).
8

Pulsa Iniciar simulación. Comprueba:

  • El semáforo arranca, hace los 5 ciclos completos (≈ 15 segundos).
  • Cuando termina, los tres LEDs quedan apagados.
  • El semáforo no vuelve a empezar (porque loop() está vacío).
🔍 Si quieres reiniciar el semáforo, pulsa Detener y Iniciar de nuevo: setup() se ejecuta otra vez.
9

Haz clic en Compartir en Tinkercad y copia el enlace público del proyecto.
