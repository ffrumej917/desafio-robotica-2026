ENTENDER — IF anidado y variable bool
⏱ 10 minutos
1

Hoy hay tres conceptos nuevos. Léelos despacio antes de seguir:

🎚 Interruptor: como el de la luz de tu casa. Cuando lo cambias de posición, queda fijo en HIGH o en LOW. No hay que mantenerlo pulsado.
🔘 Pulsador: como el timbre. Solo está en HIGH mientras lo aprietas. Al soltar vuelve a LOW.
📌 Variable bool: una variable que solo guarda dos valores: true (verdadero) o false (falso).
2

El programa tiene dos decisiones encadenadas (IF-ELSE anidado):

🔵 Decisión 1 (exterior): ¿está el interruptor en ON?
→ Si NO: todos los LEDs apagados.
→ Si SÍ: ejecuta el ciclo del semáforo, vigila el pulsador y comprueba la decisión 2.
🟣 Decisión 2 (interior): ¿se ha pulsado el botón en algún momento durante el ciclo?
→ Si NO: continúa con el ciclo normal.
→ Si SÍ: activa la secuencia de emergencia (amarillo 2 s).
3

⚠️ El truco del bool: mientras el Arduino está esperando con delay(), no puede leer el pulsador. Si el alumno pulsa el botón mientras el verde está encendido, ese instante se pierde a menos que lo "guardemos".

La solución es leer el pulsador antes de cada delay y guardar el resultado en una variable booleana llamada emergencia. Si en algún instante del ciclo se detecta que el pulsador está en HIGH, escribimos true en la variable. Al final del ciclo, comprobamos si está a true y entonces activamos la emergencia.

📌 Es como un post-it: "¿alguien ha pulsado durante el ciclo?". Si sí, lo apuntamos. Al final del ciclo miramos el post-it.

PROGRAMAR — Monta y codifica en Tinkercad
⏱ 30 minutos
6

Abre tinkercad.com → Circuits → nuevo proyecto. Monta el circuito siguiendo la imagen del paso 4. Importante: el primer botón es un slide switch (interruptor deslizante), no un pulsador. Búscalo en el panel de componentes.

⚠️ No te saltes ninguna resistencia: los 220 Ω en cada LED y los 10 kΩ pull-down en interruptor y pulsador son imprescindibles.
7

Comprueba el circuito antes de programar: 5 V y GND distribuidos por los carriles de la breadboard. Cada componente correctamente conectado.

8

Escribe el siguiente programa en el editor de código de Tinkercad:

void setup() { pinMode(13, OUTPUT); // LED verde pinMode(12, OUTPUT); // LED amarillo pinMode(11, OUTPUT); // LED rojo pinMode(6, INPUT); // interruptor (ON/OFF) pinMode(7, INPUT); // pulsador (emergencia) } void loop() { if (digitalRead(6) == HIGH) { bool emergencia = false; // Verde 1 s digitalWrite(13, HIGH); if (digitalRead(7) == HIGH) emergencia = true; delay(1000); digitalWrite(13, LOW); // Amarillo 1 s digitalWrite(12, HIGH); if (digitalRead(7) == HIGH) emergencia = true; delay(1000); digitalWrite(12, LOW); // Rojo 1 s digitalWrite(11, HIGH); if (digitalRead(7) == HIGH) emergencia = true; delay(1000); digitalWrite(11, LOW); // IF anidado: ¿se pulsó en algún momento? if (emergencia == true) { digitalWrite(12, HIGH); delay(2000); digitalWrite(12, LOW); } } else { digitalWrite(13, LOW); digitalWrite(12, LOW); digitalWrite(11, LOW); } }
9

Inicia la simulación. Comprueba los tres comportamientos:

✅ Interruptor en OFF: los 3 LEDs apagados.
✅ Interruptor en ON: ciclo verde → amarillo → rojo → verde → ... cada 1 s.
✅ Interruptor en ON + pulsar el botón en cualquier momento del ciclo: tras el rojo se enciende el amarillo durante 2 s (modo emergencia) y vuelve al ciclo.
